## Accomplishments

### 1. Cursor Hooks Fix (Windows)

Diagnosed and resolved a global Cursor IDE issue where `post-tool-use.js` opened as a tab after every file edit, shell command, and MCP execution across all project repos.

- **Root cause:** Claude-mem plugin (`thedotmack` marketplace) had installed user-level Cursor hooks at `C:\Users\cmsch\.cursor\hooks.json` with an `afterFileEdit` entry
- **Fix applied:** Removed the `afterFileEdit` hook entry; kept `beforeSubmitPrompt`, `afterMCPExecution`, `afterShellExecution`, and `stop` hooks intact
- **Also identified:** `C:\Users\cmsch\.claude-mem\hooks\post-tool-use.js` is the actual hook script file being opened
- **Scope:** Windows-only issue; macOS installs unaffected (no user-level hooks installed there)

### 3. Reusable MD-to-PDF Script

Built `scripts/md-to-pdf.mjs` — a Puppeteer-based markdown-to-PDF converter that:

- Uses Chrome (local install) via `puppeteer-core` for reliable rendering
- Loads Mermaid JS from CDN for diagram rendering
- Applies CatchFire-styled theme (gold accents, black-header tables)
- Auto-outputs to `pdf-exports/` directory
- Renders in ~6 seconds per document (vs. `npx md-to-pdf` which hung indefinitely)

## Next Steps

### HIGH Priority

- [ ] **Follower Count Filter (1K-10K range):** Add a new parameter to the creator matching and search system that restricts results to creators with 1,000 to 10,000 followers only. No large "influencer" creators should be considered or included in results. This requires:
  - Schema update: add `followerCount` field (number) to creator profile
  - Scoring update: penalize or exclude creators outside the 1K-10K band
  - API update: add `followerRange` filter to search/match endpoints
  - Scraper update: capture follower count during ingestion
  - UI update: expose follower range as a filter option

- [ ] **Align Creator Selection Criteria with Leo's 3-Category Framework:** Review and update the current scoring algorithm, LLM categorization prompts, and schema to incorporate Leo's three selection categories:
  1. **Passion/Archetype** — What drives the creator (e.g., investigative, foodie, travel, tech)
  2. **Craft/Skill** — What they excel at technically (e.g., interviewing, emotion, special FX, music)
  3. **Follower Demographics** — Who follows them (e.g., Gen Z streetwear, small-town America, retired population, car enthusiasts)
  
  The examples above are illustrative, not exhaustive. These categories need to be formalized as schema fields, integrated into the scoring algorithm, and surfaced in the UI for filtering and brief matching.

## Leo's Creator Selection Framework (New Intel)

Captured from Leo for integration into the matching algorithm:

### Three Selection Categories

| Category | Purpose | Examples (Illustrative) |
|----------|---------|------------------------|
| **Passion/Archetype** | What drives the creator | Investigative, foodie, travel, tech |
| **Craft/Skill** | Technical excellence | Interviewing, emotion, special FX, music |
| **Follower Demographics** | Audience composition | Gen Z streetwear, small-town America, retired population, car enthusiasts |

### Key Constraint: Micro-Creators Only
- **Target range:** 1,000 to 10,000 followers
- **Rationale:** CatchFire seeks craft-first storytellers, not mass-market influencers
- **Implementation:** Hard filter on follower count during search and matching
