## Infrastructure Fix: Claude-Mem Chroma

### Problem
Claude-mem's Chroma vector database server failed on every startup with exit code 1, causing a 60-second timeout before the worker fell back to "no vector search" mode. The readiness endpoint was inaccessible during this period.

### Root Cause
1. **`chromadb` Python package was not installed.** Claude-mem calls `npx.cmd chroma run` to start the Chroma server, but `chroma` is a Python CLI (not an npm package), so `npx` couldn't find it.
2. **`npx` (npm v9+) doesn't resolve PATH executables** — it only looks for npm packages.

### Fix Applied
1. Installed `chromadb` via `pip install chromadb` (Python 3.12)
2. Added Python Scripts directory to User PATH permanently
3. **Patched `worker-service.cjs`** to call `chroma.exe` directly instead of `npx.cmd chroma`:
   - Old: `r?"npx.cmd":"npx",i=["chroma","run",...]`
   - New: `r?"C:\\...\\chroma.exe":"chroma",i=["run",...]`
4. Backup saved at `worker-service.cjs.bak`

### Result
- Startup: **60s timeout + failure** → **1s + full success**
- Vector search: **disabled** → **enabled**
- MCP: connected

### Warning for Future Updates
If `claude-mem` is updated (marketplace update), the patch in `worker-service.cjs` will be overwritten. Re-apply the patch or ensure `chromadb` is accessible via `npx` (e.g., install an npm wrapper).

## Authentication System

### Strategy
Token-based Bearer authentication via `Authorization: Bearer <token>` header.

### Behavior by Environment
- **Production** (`NODE_ENV=production`): Tokens validated against `API_AUTH_TOKENS` env var (comma-separated)
- **Development** (any other `NODE_ENV`): Auth skipped, all requests pass

### Endpoint Classification
- **Public**: `/health`, `/api/v1/match`, `/api/v1/search/semantic`
- **Optional auth**: All GET endpoints (work without token, `req.authenticated` flag set)
- **Required auth**: All mutating endpoints (PATCH, POST except public ones)

### Frontend
- Login page at `/app/login` stores token in `localStorage` as `cf-auth-token`
- API client automatically includes token in all requests
