## Key Decisions

- **Combined container architecture:** Python scrapers bundled into same Docker image as Node.js backend. Scraper trigger uses `child_process.execFile` to run `python3 scraper/main.py`.
- **Internal batch import:** Scraper trigger POSTs transformed records to `localhost:PORT/api/v1/creators/batch` to reuse existing validation, dedup, and Firestore logic.
- **Datetime normalization:** Python's `datetime.isoformat()` produces microsecond precision without timezone; Zod requires millisecond precision with Z suffix. Transform truncates and appends Z.
- **`--update-env-vars` vs `--set-env-vars`:** Learned that `--set-env-vars` replaces ALL env vars on Cloud Run, wiping auth tokens. Future deploys must use `--update-env-vars` (additive).
